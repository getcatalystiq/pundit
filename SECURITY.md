# Security Policy

## Reporting a Vulnerability

If you discover a security vulnerability in Pundit, please report it responsibly.

**Do not open a public GitHub issue for security vulnerabilities.**

Instead, please email the maintainers directly with:

1. A description of the vulnerability
2. Steps to reproduce the issue
3. Potential impact
4. Any suggested fixes (optional)

## Response Timeline

- **Acknowledgment**: Within 48 hours
- **Initial assessment**: Within 7 days
- **Resolution target**: Within 30 days for critical issues

## Scope

This security policy applies to:

- The Pundit MCP server codebase
- OAuth 2.1 implementation
- Database connection handling
- API endpoints

## Out of Scope

- Vulnerabilities in dependencies (report to the respective projects)
- Issues in user-deployed infrastructure
- Social engineering attacks

## Recognition

We appreciate security researchers who help keep Pundit secure. Contributors who report valid vulnerabilities will be acknowledged in our release notes (unless they prefer to remain anonymous).
